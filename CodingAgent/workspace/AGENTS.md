# Agents in this Workspace

This OpenClaw workspace is shared by three agents that collaborate on a
**code-generation + self-correction** loop:

| Agent       | Role                                                          | Tools (allowlist) |
|-------------|---------------------------------------------------------------|-------------------|
| `coder`     | Reads `code/SPEC.md`, follows `@`-referenced skills, writes `code/solution.py`. Reads `code/DIAGNOSIS.md` for error correction. | `read`, `write`, `edit` |
| `runner`    | Executes `solution.py` (or pytest), captures the FULL Python traceback into `code/RUN_LOG.md`. | `read`, `write`, `exec` |
| `diagnoser` | On failure only: parses the traceback in `RUN_LOG.md` into an actionable Patch Plan (`code/DIAGNOSIS.md`). | `read`, `write` |

## Layout

```
workspace/
├── AGENTS.md                ← this file
├── code/
│   ├── SPEC.md              ← task definition (with @SKILL.md references)
│   ├── solution.py          ← Coder output
│   ├── RUN_LOG.md           ← Runner output (traceback verbatim)
│   ├── DIAGNOSIS.md         ← Diagnoser output (Patch Plan)
│   └── smoke_test.py        ← optional, generated by Runner
└── skills/
    ├── PYTHON_STYLE.md
    ├── ALGO_PATTERNS.md
    ├── ERROR_HANDLING.md
    └── TESTING.md
```

## Two core techniques

1. **Context Optimization** — the Coder loads ONLY the skill files referenced
   via `@FILE.md` in `SPEC.md`. Everything else stays out of the prompt.
2. **Error Correction** — when a run fails, the Diagnoser converts the raw
   Python Traceback into a Patch Plan; the Coder re-runs guided by it.
